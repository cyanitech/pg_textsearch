下面是针对这四种数组查询方式的独立示例，基于一个包含 `int[]` 类型字段的简单表。

### 准备工作（建表与示例数据）
```sql
CREATE TABLE arr_demo (
    id SERIAL PRIMARY KEY,
    numbers INT[]
);

INSERT INTO arr_demo (numbers) VALUES
    (ARRAY[1,2,3]),   -- id=1
    (ARRAY[2,3,4]),   -- id=2
    (ARRAY[4,5,6]),   -- id=3
    (ARRAY[1,3,5]);   -- id=4
```

---

### 1. `@>` (包含) —— 查询数组包含指定所有元素的记录
找出 `numbers` 字段**同时包含** 2 和 3 的记录。

```sql
SELECT * FROM arr_demo WHERE numbers @> ARRAY[2,3];
```

**结果**：  
| id | numbers |
|----|---------|
| 1  | {1,2,3} |
| 2  | {2,3,4} |

---

### 2. `<@` (被包含) —— 查询数组被包含于另一数组的记录
找出 `numbers` 的所有元素**都在** `{1,2,3,4}` 这个范围内的记录。

```sql
SELECT * FROM arr_demo WHERE numbers <@ ARRAY[1,2,3,4];
```

**结果**：  
| id | numbers |
|----|---------|
| 1  | {1,2,3} |
| 2  | {2,3,4} |

> `{4,5,6}` 中的 5、6 不在目标数组中，`{1,3,5}` 中的 5 也不在，因此不返回。

---

### 3. `&&` (重叠) —— 查询数组与目标数组有交集（至少一个公共元素）的记录
找出 `numbers` 与 `{3,4}` 有**任意公共元素**的记录。

```sql
SELECT * FROM arr_demo WHERE numbers && ARRAY[3,4];
```

**结果**：  
| id | numbers |
|----|---------|
| 1  | {1,2,3} | ← 包含 3
| 2  | {2,3,4} | ← 包含 3 和 4
| 3  | {4,5,6} | ← 包含 4
| 4  | {1,3,5} | ← 包含 3

> 本例中所有行都满足（因为每行至少含有 3 或 4 中的一个）。

---

### 4. `ANY` —— 检查某个值是否等于数组中的任一元素
找出 `numbers` 中**包含数字 3** 的记录。

```sql
SELECT * FROM arr_demo WHERE 3 = ANY(numbers);
```

**结果**：  
| id | numbers |
|----|---------|
| 1  | {1,2,3} |
| 2  | {2,3,4} |
| 4  | {1,3,5} |

> `ANY` 语法：`value = ANY(array)`。如果希望表达“value 不在数组中”，可用 `value <> ALL(array)`。
>
> --------------------字符串类型
>
> 针对字符串类型（例如 `text[]` 或 `varchar[]`），四种数组操作符的用法完全相同，只是将整数换成字符串即可。

下面使用 `text[]` 字段重新演示，并给出四种查询示例。

### 准备工作（字符串数组表）
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name TEXT,
    interests TEXT[]   -- 存储兴趣爱好，例如 'reading', 'sports'
);

INSERT INTO users (name, interests) VALUES
    ('Alice', ARRAY['reading', 'swimming', 'coding']),
    ('Bob',   ARRAY['sports', 'traveling', 'photography']),
    ('Carol', ARRAY['reading', 'sports', 'music']),
    ('David', ARRAY['coding', 'gaming']);
```

---

### 1. `@>` (包含) —— 查询数组包含所有指定字符串的记录
找出**同时喜欢 'reading' 和 'sports'** 的用户。

```sql
SELECT * FROM users WHERE interests @> ARRAY['reading', 'sports'];
```

**结果**：  
| id | name  | interests                     |
|----|-------|-------------------------------|
| 3  | Carol | {reading,sports,music}        |

> Alice 缺少 'sports'，Bob 缺少 'reading'，David 两项都没有。

---

### 2. `<@` (被包含) —— 查询数组被完全包含于另一字符串数组的记录
找出兴趣**全部都在** `{'reading','sports','music'}` 范围内的用户（即兴趣不能超出这个集合）。

```sql
SELECT * FROM users WHERE interests <@ ARRAY['reading', 'sports', 'music'];
```

**结果**：  
| id | name  | interests               |
|----|-------|-------------------------|
| 1  | Alice | {reading,swimming,coding} | ← 'swimming' 和 'coding' 不在目标集合 → **不返回** ❌
| 3  | Carol | {reading,sports,music}    | ← 全部在目标集合内 → ✅

> 实际上只有 Carol 符合。Alice 和 Bob 都有不在集合内的兴趣，David 有 'gaming' 也不符合。

---

### 3. `&&` (重叠) —— 查询数组与目标字符串数组有任意公共元素的记录
找出兴趣与 `{'sports','gaming'}` 有**交集**的用户。

```sql
SELECT * FROM users WHERE interests && ARRAY['sports', 'gaming'];
```

**结果**：  
| id | name  | interests                     |
|----|-------|-------------------------------|
| 2  | Bob   | {sports,traveling,photography} | ← 包含 'sports'
| 3  | Carol | {reading,sports,music}          | ← 包含 'sports'
| 4  | David | {coding,gaming}                 | ← 包含 'gaming'

> Alice 没有 'sports' 或 'gaming'，因此不返回。

---

### 4. `ANY` —— 检查某个字符串是否等于数组中的任一元素
找出**喜欢 'coding'** 的用户。

```sql
SELECT * FROM users WHERE 'coding' = ANY(interests);
```

**结果**：  
| id | name  | interests              |
|----|-------|------------------------|
| 1  | Alice | {reading,swimming,coding} |
| 4  | David | {coding,gaming}        |

> `ANY` 左侧可以是任意表达式，右侧为数组字段。也常用 `'coding' = ANY(interests)` 或 `'coding' IN (SELECT unnest(interests))`，但 `= ANY` 最简洁高效。

---

### ⚠️ 字符串数组注意事项
1. **大小写敏感**：`'Reading'` 与 `'reading'` 不同。如需不区分大小写，可以：
   - 存储前统一转为小写（推荐）；
   - 使用 `citext` 扩展中的 `citext[]`；
   - 在查询中用 `LOWER` 函数，但会失去索引支持。
2. **空白字符**：字符串比较时前后空格也会影响匹配，建议存储前 `trim()`。
3. **GIN 索引同样适用**：为 `interests` 字段创建 GIN 索引可加速以上所有查询：
   ```sql
   CREATE INDEX idx_users_interests ON users USING GIN (interests);
   ```
4. **ANY 与 GIN 索引**：`= ANY(array)` 通常也能受益于 GIN 索引（通过 `array_ops` 默认支持）。
