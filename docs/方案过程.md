## V6 Segment Format 实现总结

### 格式变更

| 组件 | V5 | V6 |
|---|---|---|
| Header | `TpSegmentHeader` | `TpSegmentHeaderV6`（+`positions_offset`, `capability_flags`） |
| Skip entry | `TpSkipEntry`（20B） | `TpSkipEntryV6`（28B，+`position_offset`） |
| Block data | posting only | posting + position data（varint count + delta positions） |
| Capability | 无 | `TP_SEGMENT_FLAG_HAS_POSITIONS`（0x01） |

### 修改的文件

| 文件 | 变更 |
|---|---|
| `format.h` | V6 版本号、能力位、`TpSegmentHeaderV6`、`TpSkipEntryV6` |
| `segment.h` | `tp_skip_entry_size()` 加 V6 分支 |
| `dictionary.h` | `TermInfo` 加 `positions[]` / `position_counts[]` |
| `segment.c` | V6 header 写/读、position data 写入、varint encode 工具、skip entry 回写 |
| `io.h` | `TpSegmentReader` 加 `v6_data` sidecar |
| `scan.c`（segment） | `tp_segment_read_skip_entry` 适配 V6 |
| `build.c` | `tp_do_spill` 中二次遍历 chain 收集 positions，填入 TermInfo |

### 执行流

```
spill 时:
  chain source → TermInfo[] + docmap
    → 二次遍历 chain，读取 TpVector 中的 positions
    → 填入 TermInfo.position_counts[] / positions[]

  phrase recheck 主路径 → chain positions（零 heap 访问）
  phrase recheck 回退 → heap（segment 尚不支持直接消费 positions）
```

## Merge 路径 Positions 传播 实现总结

### 修改的文件

| 文件 | 变更 |
|---|---|
| `merge_internal.h` | `TpMergePostingInfo` 加 `positions`/`position_count`；`TpPostingMergeSource` 加 `current_position_offset`；`MergeTermBlockInfo` 加 `position_offsets` |
| `merge.c` | 新增 `merge_read_varint`、`merge_write_varint_encode`、`merge_write_block_positions` 辅助函数；全量重写 `write_merged_segment_to_sink` |

### 核心逻辑

1. **V6 检测**：遍历所有 source segment，任一为 V6 且 `capability_flags & HAS_POSITIONS` → 输出 V6
2. **Header 写入**：V6 时写 `TpSegmentHeaderV6`（含 `capability_flags`）
3. **posting_source_convert_current**：V6 时从 block position stream 读取当前 doc 的 delta-encoded positions，存入 `TpMergePostingInfo`
4. **FLUSH_BLOCK 宏**：V6 时额外写入 per-doc 的 varint position data；`TpSkipEntryV6` 带 `position_offset`
5. **Skip entry**：V6 时累积 `TpSkipEntryV6`（28B），V5 时累积 `TpSkipEntry`（20B）
6. **Dict entry backpatch**：根据输出格式使用正确的 `skip_entry_size` 计算 offset
7. **Header backpatch**：V6 时回填 `positions_offset`

### 执行流

```
merge 时:
  检测各 source 是否为 V6（has positions）
    ↓
  V6? → 写 TpSegmentHeaderV6 + 用 TpSkipEntryV6
     ↓
  对每个 term:
    posting_source.load_block 时捕获 V6 position_offset
    posting_source_convert_current 时读取 block position stream
    存活 doc → block_buf + position 数组
    FLUSH_BLOCK: 写 postings + 写 positions + 记录 skip entry V6
    回写 dict entry offset（用正确的 skip entry size）
  回写 header positions_offset
```
